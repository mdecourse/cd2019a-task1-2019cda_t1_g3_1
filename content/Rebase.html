Rebase (重新定義參考基準)

重新定義某分支的參考基準。每一個 commit 都能增加分支，而分支的基準就是產生出此分支的 commit ，rebase 是能把產生出該分支的 commit 改到另一個 commit ，但是使用 rebase 會影響到 commit 原有的先後關係，所以操作時要多加注意。在此使用範例去解說：

(1) 利用 scite 建立一個 test rebase.htm ，內容如下圖並另存至資料夾。

(2) 在分支 master 上進行提交

(3) 修改先前的 test rebase.htm ，內容如下圖，在分支 master 進行第二次提交。

(4) 在 test rebase-1的 commit 中增加一個 test 分支。

(5) 切換至分支 test，在 test rebase-1 的 commit 其所有內容會回至先前的這個 commit 所擁有的內容。

 

(6) 修改 scite 中的內容，並在分支 test 中提交。

(7) 在分支 test 執行 rebase

(8) 會讓分支 master 的內容與分支 test 接再一起。
